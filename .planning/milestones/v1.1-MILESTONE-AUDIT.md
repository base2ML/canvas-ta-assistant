## Tech Debt

### Pre-existing: Nginx SSE Buffering for Posting Endpoint

**File:** `canvas-react/nginx.conf`
**Severity:** Low (does not affect development; only impacts Docker deployment with large batches)

The `/api/comments/post/{assignment_id}` SSE bulk posting route is served by the generic `/api` proxy block with `proxy_read_timeout 120s` and no `proxy_buffering off` directive. The `/api/canvas/sync` SSE route has its own dedicated block with 600s timeout and explicit SSE headers. Long posting batches in Docker could buffer or time out.

**Resolution:** Add a dedicated location block for `/api/comments/post/` with `proxy_buffering off` and a higher timeout, mirroring the canvas sync block.

This is a Phase 2/3 concern carried forward ã not introduced by Phase 4.
