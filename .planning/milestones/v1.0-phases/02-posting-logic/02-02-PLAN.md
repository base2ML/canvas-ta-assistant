**2. Add POST /api/comments/post/{assignment_id} endpoint** (add after the preview endpoint from Plan 01):

```python
@app.post("/api/comments/post/{assignment_id}")
async def post_comments(
    assignment_id: int,
    request_body: PostCommentsRequest,
    request: Request,
):
```

The endpoint returns an EventSourceResponse that streams SSE events. Implementation:

**Pre-flight validation (before SSE generator):**
- Validate posting safety: call validate_posting_safety(request_body.course_id). If unsafe, raise HTTPException(403, reason).
- Validate request has user_ids: if empty, raise HTTPException(400, "No user_ids provided").
- Resolve template via resolve_template() (skip if override_comment provided). If neither template_id/template_type nor override_comment, raise 400.
- Get assignment from db.get_assignments(request_body.course_id), find by assignment_id. Raise 404 if not found.
- Get submissions via db.get_submissions(request_body.course_id, assignment_id).
- Get users via db.get_users(request_body.course_id).
- Read max_late_days from settings via db.get_setting("max_late_days_per_assignment"), default 7.
- **Build a set of user_ids that have submissions** for quick lookup: `submission_user_ids = {s['user_id'] for s in submissions}`. This is used for the submission existence check below.

**SSE generator function (async def event_generator()):**

The generator yields dicts with "event" and "data" keys. Events:

a) **"started" event** -- emitted once at beginning:
```json
{"event": "started", "data": {"total": N, "assignment_id": X, "dry_run": bool}}
```

b) **For each user_id in request_body.user_ids:**

- Check `await request.is_disconnected()` -- if True, yield a "cancelled" event and return early
- Yield a **"progress" event**:
  ```json
  {"event": "progress", "data": {"current": idx, "total": total, "user_id": user_id, "status": "processing"}}
  ```

- **Submission existence check (SAFE-06)**: Check if `user_id in submission_user_ids`. If NOT found:
  - Log a warning via loguru: `logger.warning(f"No submission found for user {user_id} on assignment {assignment_id}, skipping")`
  - Add to skipped list with reason "no_submission"
  - Yield **"skipped" event**: `{"event": "skipped", "data": {"user_id": user_id, "reason": "no_submission"}}`
  - Continue to next user (do NOT attempt to post)

- **Duplicate check** (in-loop, per research recommendation for freshness): call `db.check_duplicate_posting(course_id, assignment_id, user_id, template_id)`. If duplicate found:
  - Add to skipped list with reason "already_posted"
  - Yield **"skipped" event**: `{"event": "skipped", "data": {"user_id": user_id, "reason": "already_posted"}}`
  - Continue to next user

- **Calculate late days data**: call `calculate_late_days_for_user(user_id, assignment, submissions, max_late_days)`

- **Render comment text**: if override_comment, use that; else call `render_template(template_text, late_days_data)`. Wrap in try/except ValueError to catch rendering errors -- add to failed list and continue.

- **Dry run check**: if `request_body.dry_run`:
  - Add to successful list (as dry_run success)
  - Yield **"dry_run" event**: `{"event": "dry_run", "data": {"user_id": user_id, "comment_text": comment_text, "variables": late_days_data}}`
  - Continue (no Canvas API call, no history recording)

- **Post to Canvas**: wrap `await asyncio.to_thread(canvas_sync.post_submission_comment, course_id, assignment_id, user_id, comment_text)` in try/except:
  - On success:
    - Record in posting history: `db.record_comment_posting(course_id, assignment_id, user_id, template_id, comment_text, status='posted', canvas_comment_id=result.get('canvas_comment_id'))`
    - Add to successful list
    - Yield **"posted" event**: `{"event": "posted", "data": {"user_id": user_id, "canvas_comment_id": result.get('canvas_comment_id')}}`
  - On failure (any Exception):
    - Record failure in posting history: `db.record_comment_posting(course_id, assignment_id, user_id, template_id, comment_text, status='failed', error_message=str(e))`
    - Add to failed list with error details
    - Yield **"error" event**: `{"event": "error", "data": {"user_id": user_id, "error": str(e)}}`
    - Log error via loguru
    - **Continue to next user** (best-effort execution per user decision)

- **Rate limiting**: after each successful post (not dry run, not skip, not error), add `await asyncio.sleep(0.5)` delay between posts. This is the 0.5-1s delay per INFRA-06. Use 0.5s as base; the exponential backoff in post_submission_comment handles 429s separately.

c) **"complete" event** -- emitted after all users processed:
```json
{
  "event": "complete",
  "data": {
    "attempted": total,
    "successful": len(successful),
    "failed": len(failed),
    "skipped": len(skipped),
    "dry_run": request_body.dry_run,
    "failed_details": [{"user_id": X, "error": "..."}, ...],
    "skipped_details": [{"user_id": X, "reason": "..."}, ...]
  }
}
```

Note: skipped_details includes BOTH "already_posted" and "no_submission" reasons so the frontend can distinguish between the two skip types.

**Return** `EventSourceResponse(event_generator())`.

**Important implementation notes:**
- All "data" values in SSE events must be JSON-serialized strings: use `json.dumps(data_dict)`
- template_id for duplicate check and history: use resolved template's id, or None if override_comment
- User name lookup: find from users list for logging, but events use user_id (frontend has user data)
- The `json` module is already imported in main.py
