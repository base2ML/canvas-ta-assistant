# Requirements: Canvas TA Dashboard - Late Day Comment Posting

**Defined:** 2026-02-15
**Core Value:** Safely and efficiently post accurate late day feedback to student submissions, preventing manual errors and saving TA time

## v1 Requirements

Requirements for initial release. Each maps to roadmap phases.

### Backend Infrastructure

- [ ] **INFRA-01**: Database table for comment templates (template_id, template_type, template_text, created_at, updated_at)
- [ ] **INFRA-02**: Database table for comment posting history (posting_id, course_id, assignment_id, user_id, template_id, comment_text, canvas_comment_id, posted_at, status)
- [ ] **INFRA-03**: Template CRUD functions in database.py (create_template, get_templates, update_template, delete_template)
- [ ] **INFRA-04**: History recording function in database.py (record_comment_posting with transaction support)
- [ ] **INFRA-05**: Canvas API integration function in canvas_sync.py (post_submission_comment with retry logic)
- [ ] **INFRA-06**: Sequential posting with rate limiting (0.5-1s delay between comments)
- [ ] **INFRA-07**: Exponential backoff retry on Canvas API 429 errors (1s, 2s, 4s delays)
- [ ] **INFRA-08**: Duplicate detection check before posting (query posting history by assignment_id + user_id)
- [ ] **INFRA-09**: Comprehensive audit logging (log all posting attempts with full comment text and metadata)

### Template Management

- [ ] **TMPL-01**: Store two message template types (penalty and non-penalty) in database
- [ ] **TMPL-02**: Pre-populate default templates with existing notebook message logic on first run
- [ ] **TMPL-03**: Template variable substitution using Python str.format() for {days_late}, {days_remaining}, {penalty_days}, {penalty_percent}, {max_late_days}
- [ ] **TMPL-04**: Template CRUD API endpoints (GET /api/templates, POST /api/templates, PUT /api/templates/:id, DELETE /api/templates/:id)
- [ ] **TMPL-05**: Template validation before saving (check for unclosed braces, validate variable names)
- [ ] **TMPL-06**: Template configuration UI in Settings page (two textarea fields with variable reference list)

### Posting Workflow

- [ ] **POST-01**: Bulk posting endpoint (POST /api/comments/post/:assignment_id with optional user_id filter)
- [ ] **POST-02**: Preview endpoint (POST /api/comments/preview/:assignment_id returns rendered comments without posting)
- [ ] **POST-03**: Preview modal in LateDaysTracking page showing penalty cases before posting
- [ ] **POST-04**: Confirmation dialog displaying course name, assignment name, and student count
- [ ] **POST-05**: Dry run mode that renders templates and validates without Canvas API calls
- [ ] **POST-06**: Error handling that continues posting on individual failures and reports detailed errors after batch completes
- [ ] **POST-07**: Progress indicator showing "Posting X/Y comments..." during bulk operation
- [ ] **POST-08**: Manual comment override capability (edit individual rendered comments before posting)
- [ ] **POST-09**: Posting history display showing posted comments with timestamps and status
- [ ] **POST-10**: Filter late day tracking view to show "Already posted" status per student/assignment

### Safety Mechanisms

- [ ] **SAFE-01**: Test mode toggle in Settings page (when enabled, uses sandbox course ID: 20960000000447574)
- [ ] **SAFE-02**: Course ID validation before posting (verify course_id matches expected value)
- [ ] **SAFE-03**: Duplicate prevention check that queries posting history before each post
- [ ] **SAFE-04**: Visual warning in UI when posting to production course (confirmation required)
- [ ] **SAFE-05**: Unique constraint in posting history table (course_id, assignment_id, user_id, template_id) to prevent DB-level duplicates
- [ ] **SAFE-06**: Submission existence check before posting (skip posting if submission doesn't exist, log warning)

### Configuration

- [ ] **CONF-01**: Max late days per assignment setting in Settings page (default: 7, configurable integer)
- [ ] **CONF-02**: Test mode configuration stored in settings table (default: false)
- [ ] **CONF-03**: Template management UI for creating, editing, and deleting templates
- [ ] **CONF-04**: Settings validation to prevent invalid configurations (e.g., negative max late days)

## v2 Requirements

Deferred to future release. Tracked but not in current roadmap.

### Enhanced Features

- **POST-ENH-01**: Undo recently posted comments (delete via Canvas API within configurable time window)
- **TMPL-ENH-02**: Template variable tester showing sample output with mock data
- **POST-ENH-03**: Multi-assignment bulk posting (post comments across multiple assignments at once)
- **TMPL-ENH-04**: Template sharing across TAs (multi-user support)
- **POST-ENH-05**: Comment attachments (attach files with posted comments via Canvas API)

### Advanced Functionality

- **TMPL-ADV-01**: Jinja2 template engine for complex conditional logic (if/else statements, loops)
- **POST-ADV-02**: Parallel posting with Canvas API request batching
- **SAFE-ADV-03**: Automated rollback on posting failures (delete successfully posted comments if batch fails)

## Out of Scope

Explicitly excluded. Documented to prevent scope creep.

| Feature | Reason |
|---------|--------|
| Automated scheduling of comment posting | Manual trigger only - TAs should review data before posting |
| Comment editing after posting | Use Canvas UI directly - modifying historical comments is risky |
| Email notifications to students | Canvas handles this natively when comments are posted |
| Rich text formatting in comments | Plain text only - reduces complexity and testing burden |
| Multi-course bulk operations | Posting is per-course only - prevents accidental cross-course posting |
| Comment templates with attachments | Defer to v2 - file handling adds significant complexity |
| Real-time collaborative editing of templates | Single-user local deployment - no multi-user support |
| Integration with external grading systems | Canvas-only integration - expanding scope too broadly |
